# Security Policy

## Reporting
If you discover a security issue, do not open a public issue first.

Report privately with:
- Affected file/module
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

## Scope Notes
- This repository includes research and experimental code.
- High-risk findings include data leakage, arbitrary code execution, and supply-chain issues.

## Response
- Acknowledge report within 3 business days.
- Provide triage status and remediation plan.
- Publish fix notes after a patch is available.
